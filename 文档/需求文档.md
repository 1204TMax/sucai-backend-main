# 工作流后台需求文档

## 一、项目定位

`sucai-backend-main` 是独立后台项目，用于管理量化投放系统中的 ComfyUI 封装节点和节点类型配置。

后台项目与主站 `sucai-main` 并列，不属于主站子目录。主站后续只需要在 `AI 创作 - 项目管理` 下新增「工作流后台」入口，点击后跳转到独立后台地址。本次后台项目不修改主站。

## 二、页面范围

| 页面 | 文件 | 说明 |
| --- | --- | --- |
| 登录页 | `login.html` | 后台登录入口，无注册；原型阶段不接真实中台账号，但需要展示账号、密码空值提示 |
| 后台框架 | `index.html` | 左侧菜单布局，包含工作流管理和工作流类型管理两个入口 |
| 工作流管理 | `工作流管理.html` | 管理 ComfyUI JSON 封装出的节点 |
| 工作流类型管理 | `配置管理.html` | 管理节点类型配置 |

## 三、后台结构

后台登录后只需要左侧菜单和内容区。

左侧菜单包含两个功能入口：

- 工作流管理
- 工作流类型管理

后台和主站完全独立，不复用主站登录态，不依赖主站页面状态。

### 3.1 登录规则

| 字段 / 控件 | 规则 | 校验 / 交互 |
| --- | --- | --- |
| 账号 | 使用中台账号登录 | 账号不能为空 |
| 密码 | 使用中台账号密码登录 | 密码不能为空 |
| 登录按钮 | 点击后进入后台框架 | 校验失败时停留在登录页并提示原因 |

权限规则：

- 后台使用中台账号登录。
- 后台自行做权限管理。
- 本期所有用户都可以查看后台内容。

### 3.2 后台框架规则

| 区域 | 规则 |
| --- | --- |
| 左侧菜单 | 固定展示「工作流管理」「工作流类型管理」两个入口 |
| 默认页面 | 登录后默认进入工作流管理 |
| 选中态 | 当前菜单需要展示选中态 |
| 用户信息 | 左下角展示当前用户名称和登出按钮 |
| 登出 | 点击后返回登录页 |

## 四、工作流管理

### 4.1 页面定位

工作流管理用于把开发者本地 ComfyUI 跑通并导出的 JSON 封装为系统可用节点。

### 4.2 节点状态

节点只有两个状态：

- 已上线
- 已下线

新建节点保存成功后默认状态为「已下线」，需要手动上线后才进入主站节点库。

### 4.3 节点操作

节点支持：

- 新建
- 上线
- 下线
- 编辑
- 删除

规则：

- 已上线节点也可以编辑。
- 已上线节点也可以删除。
- 编辑、删除、上线、下线不影响已经开始执行的工作流任务。
- 节点删除或下线后，已发布工作流中引用该节点的位置不允许继续使用；节点右上角用红色字体提示：**该节点被删除/下线，无法使用**。
- 新开任务运行到已删除或已下线节点时，任务报错：**XX节点：已被删除/下线**。
- 系统需要记录节点编辑日志，但日志不展示在页面上。

日志记录内容：

- 操作人
- 操作类型
- 操作时间
- 节点 ID
- 节点名称
- 操作前后状态
- ComfyUI JSON 版本标识

操作反馈：

| 操作 | 确认规则 | 成功反馈 |
| --- | --- | --- |
| 上线 | 需要二次确认 | 成功后 Toast 提示，并刷新列表状态和最近编辑时间 |
| 下线 | 需要二次确认 | 成功后 Toast 提示，并刷新列表状态和最近编辑时间 |
| 删除 | 需要二次确认 | 成功后 Toast 提示，并从列表移除 |

### 4.4 列表字段

| 字段 | 展示规则 | 数据来源 | 交互 / 校验 |
| --- | --- | --- | --- |
| 封面 | 展示节点封面图；未上传时展示默认占位封面 | 新建 / 编辑节点时用户上传 | 图片必须为 1:1；用于后台列表和主站节点库展示 |
| 名称 | 展示节点名称，不展示输出说明 | 新建 / 编辑节点时用户输入 | 不超过 10 个字；列表中作为主识别字段展示 |
| 描述 | 展示节点描述 | 新建 / 编辑节点时用户输入 | 不超过 50 个字；列表中不展示输入项标签 |
| 节点类型 | 展示节点所属类型 | 工作流类型管理中的节点类型配置 | 用于工作流编辑页节点库分组 |
| 输出类型 | 展示该封装节点最终输出类型 | 新建 / 编辑节点时选择的输出项 | 当前展示为文本、图片、视频等产品化类型，不展示 ComfyUI 原始字段名 |
| 状态 | 展示「已上线」或「已下线」标签 | 节点状态 | 只有已上线、已下线两种状态 |
| 创建人 | 展示节点创建人姓名 | 系统记录 | 原型阶段使用 mock 数据 |
| 最近编辑时间 | 展示最近一次编辑时间 | 系统记录 | 编辑、上线、下线后更新时间 |
| 操作 | 展示编辑、上线/下线、删除 | 节点状态 | 已上线节点也可以编辑和删除；删除需要二次确认 |

### 4.4.1 字段类型与输出类型映射

后台需要将 ComfyUI 字段类型映射为主站可理解的产品化类型。该映射是类型匹配的 SSOT。

| ComfyUI 字段类型示例 | 产品化类型 | 是否数组 | 主站用途 |
| --- | --- | --- | --- |
| `STRING` / `TEXT` | 文本 | 否 | 文案输入、提示词输入、文本结果 |
| `STRING[]` / `TEXT[]` | 文本 | 是 | 多条文案结果或文本列表 |
| `IMAGE` | 图片 | 否 | 单张图片输入或输出 |
| `IMAGE[]` | 图片 | 是 | 多张图片结果 |
| `VIDEO` | 视频 | 否 | 单个视频输入或输出 |
| `VIDEO[]` | 视频 | 是 | 多个视频结果 |
| 其他已知文件类字段 | 文件 | 按字段实际结构判断 | 通用文件输入或输出 |
| 未知类型 | 文件 | 否 | 兜底为文件类型，并在后续开发中补充映射 |

类型匹配规则：

- 主站输入来源选择时，优先按产品化类型匹配。
- 产品化类型不同的输出，不允许作为输入来源。
- 是否数组需要保留为字段属性；若后续节点只支持单值输入，则数组输出不可直接作为来源。
- 列表中的「输出类型」展示产品化类型，不展示 ComfyUI 原始字段类型。

### 4.5 顶部筛选

| 控件 | 规则 | 交互 |
| --- | --- | --- |
| 搜索框 | 支持搜索节点名称和节点描述 | 输入后筛选列表；原型阶段仅筛选 mock 数据 |
| 节点类型筛选 | 下拉展示全部节点类型 | 选中后仅展示该类型节点 |
| 状态筛选 | 下拉展示全部状态、已上线、已下线 | 选中后仅展示该状态节点 |
| 新建节点按钮 | 点击打开新建节点弹窗 | 不跳转页面 |

### 4.6 新建节点

点击「新建节点」打开弹窗。

基础信息：

| 字段 | 类型 | 规则 | 校验 |
| --- | --- | --- | --- |
| 节点名称 | 文本输入 | 用户自行输入，用于节点列表和主站节点库展示 | 必填；不超过 10 个字；不允许使用“审核”“开始”“结束”，如果输入则提示：**名称不合法，请更换** |
| 节点描述 | 多行文本 | 用户自行输入，用于说明节点用途 | 非必填；不超过 50 个字 |
| 节点类型 | 下拉选择 | 选项来自工作流类型管理中的节点类型配置 | 必填 |
| 节点封面图 | 图片上传 | 每个封装节点最多上传一张封面图，用于后台列表和主站节点库展示；未上传时使用默认占位封面 | 非必填；仅支持 PNG、JPG、WEBP；图片宽高比例必须为 1:1；不符合时在上传区域下方提示 |

ComfyUI JSON 上传：

| 字段 / 状态 | 规则 |
| --- | --- |
| 上传入口 | 在新建 / 编辑弹窗中上传 ComfyUI JSON |
| 文件格式 | 仅允许上传 `.json` 文件 |
| 文件大小 | 单个 JSON 文件大小不超过 100M |
| 上传组件 | 使用文件上传组件选择本地 JSON 文件，不展示进度条，不需要单独上传列表 |
| 解析结果 | 上传完成后在上传组件内展示解析成功状态 |
| 上传失败 | 上传失败或解析错误时，在上传区域下方展示失败原因 |
| 固定错误类型 | 包括：非 `.json` 文件、文件超过 100M、上传中断、JSON 解析失败、解析成功但无可选节点 / 字段 |
| 替换 JSON | 新建或编辑时允许重新上传并替换 JSON；替换后需要重新校验已选择的输入项和输出项 |
| 替换后的字段处理 | 替换 JSON 后，如果原已选 ComfyUI 节点或字段匹配不到，则清空对应选择；保存时按必填规则重新校验 |
| 生效时机 | 替换 JSON 后，只有点击保存并校验通过才正式生效 |
| 原型范围 | 原型阶段使用 mock 上传状态和 mock 解析结果，不做真实 JSON 解析 |

输入项：

| 字段 | 类型 | 规则 | 校验 |
| --- | --- | --- | --- |
| ComfyUI 节点 | 下拉选择 | 从上传 JSON 解析出的 ComfyUI 节点中选择；原型阶段使用 mock 节点 | 必填 |
| 输入字段 | 下拉选择 | 选择当前 ComfyUI 节点下的输入字段；切换节点后字段选项随之变化 | 必填 |
| 展示名称 | 文本输入 | 配置该输入项在主站工作流编辑页中的展示名称 | 必填；不超过 10 个字；同一个封装节点内不允许完全重复 |
| 描述 | 文本输入 | 说明该输入项用途，辅助主站使用者理解 | 非必填；不超过 50 个字 |

补充规则：

- 输入项允许新增多个。
- 输入项允许删除。
- 输入项按配置顺序展示；主站工作流编辑页也按该顺序展示。
- 输入项默认必填，不提供“是否必填”配置。
- 输入项不提供默认值配置。
- 所选输入字段的内部类型和产品化输入类型由系统解析并保存，用于主站类型匹配，不在后台表单中作为独立字段展示。

输出项：

| 字段 | 类型 | 规则 | 校验 |
| --- | --- | --- | --- |
| ComfyUI 节点 | 下拉选择 | 从上传 JSON 解析出的 ComfyUI 节点中选择；原型阶段使用 mock 节点 | 必填 |
| 输出字段 | 下拉选择 | 选择当前 ComfyUI 节点下的输出字段；切换节点后字段选项随之变化 | 必填 |
| 输出类型 | 自动展示 | 展示为文本、图片、视频等产品化类型；用于列表展示和主站类型匹配 | 不允许手动编辑 |
| 输出展示名称 | 文本输入 | 配置该封装节点最终输出在主站中的展示名称 | 必填；不超过 10 个字 |

补充规则：

- 输出项只能配置一个。
- 输出项只选择某个 ComfyUI 节点自身暴露的输出字段，不做更复杂的“节点输出再取节点”链路。
- 所选输出字段的内部类型由系统解析并保存，用于映射产品化输出类型，不在后台表单中作为独立字段展示。

保存校验：

- 节点名称不能为空，且不超过 10 个字。
- 节点名称不允许使用“审核”“开始”“结束”；如果输入，提示：**名称不合法，请更换**。
- 节点描述不超过 50 个字。
- 必须选择节点类型。
- 如果上传节点封面图，必须是 1:1 图片。
- 必须上传并解析 ComfyUI JSON。
- 至少配置 1 个输入项。
- 每个输入项必须选择 ComfyUI 节点。
- 每个输入项必须选择该节点下的输入字段。
- 每个输入项必须填写展示名称。
- 每个输入项展示名称不超过 10 个字。
- 每个输入项描述不超过 50 个字。
- 同一封装节点内，输入项展示名称不能完全相同。
- 必须选择输出 ComfyUI 节点。
- 必须选择输出字段。
- 必须填写输出展示名称。
- 输出展示名称不超过 10 个字。
- 校验失败时，不允许保存，弹窗保持打开，并在弹窗内展示错误信息，由配置人自行修改。

### 4.7 编辑节点

编辑弹窗与新建弹窗结构一致。

规则：

- 已上线节点也允许编辑。
- 保存后只影响后续新任务使用的节点配置。
- 不影响已经开始执行的任务。

### 4.8 删除节点

- 已上线和已下线节点都可以删除。
- 删除前必须二次确认。
- 删除不影响已经开始执行的任务。

## 五、工作流类型管理

### 5.1 页面定位

工作流类型管理当前只维护节点类型配置。

节点类型用于：

- 工作流管理中新建节点时的「节点类型」下拉选项。
- 主站工作流编辑页节点库分组。

### 5.2 类型列表字段

| 字段 | 展示规则 | 数据来源 | 交互 / 校验 |
| --- | --- | --- | --- |
| 类型名称 | 展示节点类型名称 | 新增 / 编辑类型时用户输入 | 必填；用于工作流管理中的节点类型下拉 |
| 类型描述 | 展示类型用途说明 | 新增 / 编辑类型时用户输入 | 非必填 |
| 创建人 | 展示类型创建人姓名 | 系统记录 | 原型阶段使用 mock 数据 |
| 最近编辑时间 | 展示最近一次编辑时间 | 系统记录 | 新增 / 编辑后更新时间 |
| 操作 | 展示编辑、删除 | 类型数据 | 删除前需要二次确认 |

### 5.3 类型操作

| 操作 | 规则 |
| --- | --- |
| 新增 | 打开新增类型弹窗，填写类型名称和类型描述 |
| 编辑 | 打开编辑类型弹窗，允许修改类型名称和类型描述 |
| 删除 | 删除前需要二次确认 |

类型删除规则：

- 如果已有节点正在使用该节点类型，不允许删除。
- 删除失败时提示：**当前类型已被节点使用，无法删除**。
- 未被节点使用的类型删除后，不再出现在新建 / 编辑节点的节点类型下拉中。
- 已有节点不会出现“无类型”状态，因为被引用的类型不允许删除。

### 5.4 类型表单字段

| 字段 | 类型 | 规则 | 校验 |
| --- | --- | --- | --- |
| 类型名称 | 文本输入 | 用于节点封装时选择节点类型，也用于主站节点库分组名称 | 必填；不超过 10 个字；不允许重复；不允许使用“审核”“开始”“结束” |
| 类型描述 | 多行文本 | 说明该类型适合承载的节点能力 | 非必填；不超过 50 个字 |

补充规则：

- 类型名称前后空格保存时自动去除。
- 类型编辑成功、删除成功后均需要 Toast 提示。
- 节点类型在主站节点库中的分组顺序按工作流类型管理列表顺序展示。

### 5.5 示例类型

- 图片处理
- 视频生成
- 文案处理
- 素材合成

## 六、任务快照规则

任务执行必须按任务开始时的工作流和节点状态快照执行。

示例：

- A 时刻，用户基于某个工作流开始任务 1。
- 1 分钟后，后台编辑了其中某个节点。
- 随后用户又开始任务 2。

规则：

- 任务 1 仍按 A 时刻的工作流节点状态执行。
- 任务 2 按第二次开始任务时的工作流节点状态执行。
- 后续节点编辑不影响已经开始的任务。
- 节点删除或下线不影响已经开始执行且已经进入快照的任务。
- 对于尚未开始的新任务，如果工作流中引用了已删除或已下线节点，允许提交任务；系统按任务开始时快照记录该节点状态，运行到该节点时失败，并报错：**XX节点：已被删除/下线**。
- 主站工作流编辑页需要在该节点右上角用红色字体提示：**该节点被删除/下线，无法使用**。

任务快照包含：

- 主站工作流连线关系。
- 主站工作流节点实例配置。
- 后台封装节点名称、描述、节点类型、封面图和状态。
- 后台封装节点对应的 ComfyUI JSON。
- 后台封装节点输入项配置。
- 后台封装节点输出项配置。
- 输入字段对应的内部类型、产品化输入类型、输出字段对应的内部类型和输出类型。

## 七、运行报错规则

如果工作流运行时 ComfyUI 报错，需要在任务查看详情中暴露 ComfyUI 原始报错信息。

展示格式：

```text
XXX节点（用户自己输入的节点名称）错误：XXXXX（ComfyUI 原始报错）
```

其中 `XXX节点` 优先使用主站画布中的节点实例名称；若主站节点实例名称为空，则使用后台封装节点名称。

后台原型不实现真实运行，只在页面和需求中保留该规则。

## 八、主站工作流编辑页补充规则

这些规则由后台封装节点提供基础数据，后续在主站工作流编辑页使用。

| 后台字段 / 规则 | 主站使用方式 |
| --- | --- |
| 节点类型 | 用于工作流编辑页节点库分组 |
| 节点名称、节点描述 | 用于节点库卡片和画布节点展示 |
| 输入项展示名称、描述、输入类型 | 用于生成封装节点右侧配置面板 |
| 输出展示名称、输出类型 | 用于下游节点输入来源选择和结束节点最终输出选择 |
| 输入类型 / 输出类型 | 用于主站做类型匹配；类型不匹配时，不能选择为输入来源 |
| 节点状态 | 已删除 / 已下线节点在主站工作流编辑页提示不可用 |

主站补充规则：

- 必填输入项必须有来源。
- 结束节点必须选择最终输出。
- 人工审核节点默认审核上一级节点输出内容。
- 开始节点仍由用户自己决定输入项。
- 不做自动映射，只做校验。
- 节点库不再展示“大模型”分组。
- 节点库不再固定展示文本生成、图片生成、视频生成节点。
- 节点库按后台配置的节点类型聚合展示封装节点。
- 人工审核保留在“审核”分组。
- 开始节点和结束节点是画布默认节点，不作为普通节点卡片展示。
