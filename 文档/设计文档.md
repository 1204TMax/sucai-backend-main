# 工作流后台设计文档

## 一、设计目标

后台是面向开发者和运营配置人员的工具系统，视觉应紧凑、清晰、专业，优先支持高频管理、筛选、编辑和确认操作。

## 二、基础规范

- 页面外层 `body padding` 使用 `8px`。
- 大模块之间 `gap` 使用 `8px`。
- 控件高度统一为 `32px`。
- 基础字号使用 `13px`。
- 卡片和面板圆角不超过 `8px`。
- 不做营销式首页。
- 不做大 hero。
- 不使用 `height: 100vh` 制造双滚动条。
- 不使用复杂动画。

## 三、颜色

采用 Ant Design 5 风格：

- 主色：`#1677ff`
- 页面背景：`#f5f7fa`
- 面板背景：`#ffffff`
- 主文字：`#262626`
- 次级文字：`#595959`
- 辅助文字：`#8c8c8c`
- 边框：`#d9d9d9`
- 细分割线：`#f0f0f0`

实际页面中通过 `共享样式/backend.css` 中的 CSS 变量统一维护。

## 四、组件

### 按钮

- 高度 32px。
- 主按钮使用蓝色背景。
- 危险操作使用红色文字或红色边框。
- 删除类操作必须二次确认。

### 输入框和下拉框

- 高度 32px。
- 圆角 6px。
- 聚焦时使用蓝色描边和浅蓝 focus ring。

### 表格

- 用于工作流节点和配置项列表。
- 表头使用浅灰背景。
- 行 hover 使用极浅灰背景。
- 操作区使用文字按钮，避免视觉过重。

### 弹窗

- 用于新建、编辑、删除确认。
- 弹窗宽度根据内容控制。
- 表单字段使用两列布局，窄屏降为单列。

### 状态标签

- 已上线：绿色标签。
- 已下线：灰色标签。
- 必填输入项：黄色标签或必填标识。

### 可排序配置项

- 输入项和输出结果以卡片形式承载。
- 卡片顶部展示自动序号，如「输入项 1」「输出结果 1」。
- 排序入口使用 icon 拖拽把手，不使用文字按钮。
- 拖拽中状态使用浅蓝背景和主色描边，保持操作反馈清晰。

## 五、页面布局

### 登录页

居中登录卡片，包含系统名称、账号、密码、登录按钮和原型提示。

### 后台主框架

- 左侧菜单宽度 220px。
- 右侧为顶部说明和 iframe 内容区。
- 菜单仅包含工作流管理和工作流类型管理。

### 工作流管理

- 顶部为标题和新建按钮。
- 下方为搜索、类型筛选、状态筛选。
- 主体为节点表格。
- 新建/编辑节点使用弹窗承载复杂表单。

### 工作流类型管理

- 顶部为标题和新增类型按钮。
- 主体为类型配置表格。
- 新增/编辑使用弹窗。

## 六、原型边界

页面只展示后台产品形态和关键交互。

不做真实接口、真实上传、真实 ComfyUI JSON 解析、真实保存、真实跨页面联动或真实运行。
